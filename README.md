In this course, the main problem I was working to solve was building an advising assistance program that could store and organize course information in a way that made it easy to search and use. The goal was to help academic advisors quickly find courses and understand their prerequisites without having to dig through unorganized data. This meant I had to think not just about getting the program to work, but also about how efficiently it could handle and retrieve information.

My approach to solving this problem really came down to understanding data structures and why they matter. I learned that the way you store data directly affects how fast and easy it is to use later. For example, something like a vector is simple and easy to work with, but structures like hash tables or binary search trees can make searching much faster and more efficient. Taking the time to understand these differences helped me make better decisions when designing my program.

I definitely ran into some roadblocks along the way, especially when working with more complex structures like hash tables and binary search trees. There were times when my program wouldn’t return the correct data or wouldn’t work at all. To get past that, I had to slow down and break things into smaller pieces. Writing out pseudocode helped me map out my logic before coding, and testing small parts of the program at a time made it easier to catch mistakes. A lot of it came down to being patient and not trying to fix everything at once.

This project changed the way I think about designing software. Before, I would usually jump straight into coding, but now I understand how important it is to plan things out first. Thinking about how the program will grow, how data will be used, and how everything connects makes a big difference. It also helped me see how different parts of a program work together instead of just focusing on one piece at a time.

It also improved how I write my code overall. I pay more attention now to making my code readable and organized so that it makes sense not just to me, but to anyone else who might look at it later. I’ve started to rely more on clear structure, meaningful function names, and separating different parts of the program into smaller sections. That makes it easier to update, fix, or expand later without breaking everything. Overall, this project helped me grow not just in coding, but in how I think about building programs in a smarter and more structured way.# CS300
